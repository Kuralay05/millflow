# Описание базы данных MillFlow

## Используемые технологии

В качестве базы данных используется MongoDB, а для работы с ней в приложении применяется Mongoose.

## Коллекции

### User

Хранит данные пользователей системы: ФИО, email, пароль и роль.

### Supplier

Содержит информацию о поставщиках зерна: наименование, контактное лицо, телефон и адрес.

### GrainBatch

Описывает партии зерна, поступающие на предприятие: номер партии, поставщик, тип зерна, масса, влажность, дата приемки, статус качества и примечание.

### ProductionBatch

Содержит данные о производственных партиях: номер партии, ссылка на партию зерна, дата производства, объем произведенной муки, отходы, статус и комментарии.

### InventoryItem

Используется для хранения информации о складских остатках: наименование продукта, тип, номер партии, количество, местоположение, статус остатка и дата обновления.

### Shipment

Хранит сведения об отгрузках: номер, клиент, продукт, партия, количество, дата и статус доставки.

### AuditLog

Фиксирует действия пользователей: тип действия, сущность, идентификатор сущности, email пользователя, время и описание действия.

## Поля и их назначение

- `batchNumber` и `shipmentNumber` используются как удобные бизнес-идентификаторы
- `supplierId` в `GrainBatch` связывает партию зерна с поставщиком
- `grainBatchId` в `ProductionBatch` связывает производство с сырьем
- `status` отражает текущее состояние сущности
- `timestamp`, `createdAt`, `updatedAt` обеспечивают контроль времени событий

## Связи между сущностями

- один поставщик может иметь много партий зерна
- одна партия зерна может быть связана с одной или несколькими производственными партиями
- складские позиции и отгрузки ссылаются на номера партий как на бизнес-идентификаторы
- журнал действий хранит историю изменений по основным сущностям системы

## Почему выбраны MongoDB и Mongoose

MongoDB удобна для учебного проекта, так как позволяет быстро развернуть базу, хранить данные в виде документов и гибко расширять структуру при необходимости. Mongoose упрощает создание схем, моделей, связей и валидации, что делает код более читаемым и удобным для объяснения на защите.
